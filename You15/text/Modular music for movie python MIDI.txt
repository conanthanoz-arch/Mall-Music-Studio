1. The MIDI Note Frequency Formula
To have Python generate raw audio waves or write precise MIDI files, your code must calculate the exact frequency (f) of a note based on its MIDI number (n).
The standard mathematical formula relative to A₄ (MIDI note 69, 440 Hz) is:
$$f = 440 \times 2^{\frac{n - 69}{12}}$$

3. Programmatic Velocity & Humanization Math
To prevent your self-generated beats from sounding sterile or robotic, apply a randomized Gaussian distribution curveto your note velocities (volume) and timing offsets.

	•	Robotic Beat: Every hi-hat hits at a fixed velocity of 127 and exactly every 0.25 seconds.
	•	Humanized Beat: Python shifts the velocity slightly using a mathematical variance:
$$\text{Velocity} = \text{Base Velocity} + \mathcal{N}(\mu=0, \sigma^2=5)$$ 
import random

def humanize_note_event(base_velocity=90):
    # Generates a random float using a normal distribution
    # Keeps velocity fluctuations natural, between roughly -5 and +5 volume points
    variance = random.gauss(0, 2.5)
    final_velocity = int(base_velocity + variance)
    
    # Clip values to legal MIDI boundaries (1-127)
    return max(1, min(127, final_velocity))
